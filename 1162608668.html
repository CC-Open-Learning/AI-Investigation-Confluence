AI Investigation : O'Reilly API (PAID)

Created by Aerish Gaba (Unlicensed), last modified on Jun 10, 2024

This page provides a complete reference to the Search v2 API, including all possible parameters and a full response that includes results, facets, and metadata. The Search API has many powerful and flexible feature, however some come with a higher cost. Most often this means slower responses as the API has to do more computation to provide the full results. The performance suggestions section provides a number of tips to make the Search API work accurately and responsively for your use case.

AUTHENTICATION:- The three most common means of authentication are

  • Browser-based authentication, with JWTs (JSON Web Tokens).

  • OAuth Token authentication, using the OAuth flow.

  • API Token based authentication.

As we are external user we are most likely to use API based authentication.

Authenticating with an API Token

Query - Use the Authorization header to provide a pre-supplied API Token to the Search API for authentication.

curl https://learning.oreilly.com/api/v2/search/?query=python -H 'Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b'

If you need an API token, use the token creator on the Integrations page of your Admin Console to generate the required search API token.

Authenticate with the API for a different rate-limiting speed (60 requests per minute, as opposed to 15 requests per minute while anonymous). To do so, request an access token from Oâ€™Reilly. After you have an access token, include an Authorization header in every request that you wish to be authenticated.

After reaching the rate-limit threshold, all API responses will return an HTTP 429 Too Many Requests response, with the message "Request was throttled. Expected available in <n> seconds." where n is the number of seconds until the user can make non-rate limited API calls again.

Performance Suggestions

  • Request the smallest set of result fields possible

  • Turn off faceting

  • Disable highlighting

  • Disable reporting of extended publisher data

  • Respect the previous and next response values

  • Prefer the formats parameter over the include_<format> parameters

  • Re-use connections as much as possible, especially HTTPS connections

  • If you are requesting more than 200 results total, you set the results per page to 200 using command {$ curl https://learning.oreilly.com/api/v2/search/?limit=200}, in order to decrease the amount of total requests being sent.

There are three fields that sound very similar:

format, content_format, and content_type.

  • format is the value that represents what kind of O'Reilly product the result represents.

  • content_format is a field that describes the content's format type at a more granular level than the format field.

  • content_type is a holdover from pre-video days, and represents our search engine's relationship between documents. A document with the content_type of 'book' actually means a parent, and 'chapter' actually means child.

  • Example - Assuming tuples of (format, content_format, content_type):

    • A book on the Platform that renders as text, not PDF, is (book, book, book).

    • A video is (video, video, book).

    • An audiobook (audiobook, audiobook, book).

    • An article, which is typically a very short EPUB (book, article, book).

    • A chapter from a reflowable EPUB-based book is (book, book, chapter).

    • A video clip is (video, video, chapter).

Most documents in the search index are updated in near real-time, relative to when the platform processes an addition or an update to a work. This is true for books, videos, audiobooks, Playlists, and Learning Paths.

Other result formats, like Live Online Trainings are updated on a regular schedule. In all cases, each result format is checked for additions, updates, and removals at least once per day.

Queries for Future Purpose

Searching:-

Use the query param to tell the Search API what search terms you're interested in. The Search API will analyze your query and return all relevant documents.

$ curl https://learning.oreilly.com/api/v2/search/?query=python

Query param can be -

  1. archive_id:

     $ curl https://learning.oreilly.com/api/v2/search/?query=archive_id:9780132823159

  2. Live Online Training via archive_id:

     $ curl "https://learning.oreilly.com/api/v2/search/?query=archive_id:0636920394075.COURSE&include_courses=true"

    Note that the URL string needs to be in quotes for this to work correctly, due to the .COURSE suffix for the identifier. Due to internal implementation requirements, any query for Live Online Trainings must include include_courses=true as one of the query params. This param should not be confused with the O'Reilly marketing term for Video Courses which can be found by passing the video_classifications=structured-learning param.

  3. Book via web_url:

     $ curl https://learning.oreilly.com/api/v2/search/?query=web_url:/library/view/the-long-tail/9780132823159/

  4. Live Online Training via web_url:

     $ curl "https://learning.oreilly.com/api/v2/search/?query=web_url:/live-training/courses/nonlinear-machine-learning-for-algorithmic-trading/0636920394075&include_courses=true"

  5. Author via query:

     $ curl https://learning.oreilly.com/api/v2/search/?query=author:\"robert martin\"

    or

     $ curl https://learning.oreilly.com/api/v2/search/?query=author:robert%20martin

    Note the enclosing and escaped quotes around "robert martin" or the alternative %20 url-encoded space character. Either are required for multiple word queries

  6. Robert Martin (author) and Clean Code (title) via query:

     $ curl https://learning.oreilly.com/api/v2/search/?query=author:robert%20martin%20title:clean%20code

  7. Publisher via query:

     $ curl https://learning.oreilly.com/api/v2/search/?query=publisher:apress
Quotes.

Sometimes, wrapping a multiple word search query in quotes improves the results returned, because that specifies to the search engine that all matches should have the terms in the query in the exact order specified.

$ curl https://learning.oreilly.com/api/v2/search/?query="microservices%20in%20python"
Boolean logic (AND, OR, NOT)

AND, OR and NOT can be used to apply boolean logic to the search results.

  • Search for Java titles that don't have Javascript (java NOT javascript)

      $ curl https://learning.oreilly.com/api/v2/search/?query=java%20Not%20javascript

  • Search for content specific to react hooks (react AND hooks)

      $ curl https://learning.oreilly.com/api/v2/search/?query=react%20AND%20hooks

Note that the terms MUST be capitalized to have effect

Parenthesis

Parenthesis can be used to group terms and combined with Boolean logic

  • Search for angular framework that excludes React (framework AND (angular NOT react))

      $ curl https://learning.oreilly.com/api/v2/search/?query=framework%20AND%20(angular%20NOT%20react)

Use the field parameter to tell the Search API which field you want your query applied to. Accepted field values are: authors, isbn, publishers, and title. Note that if field=isbn, this implicitly filters on both ISBN and archive_id

$ curl https://learning.oreilly.com/api/v2/search/?query=9780470414637&field=isbn

Filtering - Adding the filters below in the search query filters the results.

/api/v2/search/{?query,publishers,formats,topic_uuids,topics,languages,sharing_status,expert_playlist,video_classifications,certification_vendor_name,certification_name}

Examples -

  1. Filtering a query by books and videos:

     $ curl https://learning.oreilly.com/api/v2/search/?query=python&formats=book&formats=video

  2. Looking for all O'Reilly Video Course videos:

     $ curl https://learning.oreilly.com/api/v2/search?formats=video&video_classifications=structured-learning

  3. Filtering a query by Amazon Web Services (AWS) topic

      $ curl https://learning.oreilly.com/api/v2/search/?query=python&topics=Amazon%20Web%20Services%20(AWS)

  4. Filtering a query by 4 stars and above:

     $ curl https://learning.oreilly.com/api/v2/search/?query=python&average_rating__gte=3.5

    Note - the query param is average_rating__gte. Due to the internal implementation details, a four star rating would require a filter value of 3.5

    The following is the mappings used:

    1. average_rating__gte=0.5 -> All ratings from 1 to 5

    2. average_rating__gte=1.5 => All ratings from 2 to 5

    3. average_rating__gte=2.5 => All ratings from 3 to 5

    4. average_rating__gte=3.5 => All ratings from 4 to 5

    5. average_rating__gte=4.5 => All 5-star ratings

  5. Filtering a query for Spanish books:

     $ curl https://learning.oreilly.com/api/v2/search/?query=python&formats=book&languages=es

  6. Looking for all early release books

     $ curl https://learning.oreilly.com/api/v2/search/?sort=publication_date&formats=book

  • Highlighting:- Use the highlight parameter to tell the Search API that you want highlighted snippets that represent search query matches in results.

$ curl https://learning.oreilly.com/api/v2/search/?query=python&highlight=true

  • Sort & Order:- Use the sort and order parameters to tell the Search API how you want your results sorted.

$ curl https://learning.oreilly.com/api/v2/search/?query=python&sort=publication_date&order=asc

Examples

  1. Query for most recent Python books

    $ curl https://learning.oreilly.com/api/v2/search/?query=python&formats=book&sort=publication_date

  2. Query for Python DevOps videos that were recently added to the platform:

     $ curl https://learning.oreilly.com/api/v2/search/?query=python&topics=devops&formats=video&sort=date_added

  3. Look for highest rated Apress books:

     $ curl https://learning.oreilly.com/api/v2/search/?publisher=apress&formats=book&sort=rating

  4. Find all popular Expert Playlists

     $curl https://learning.oreilly.com/api/v2/search/?include_collections=true&formats=collection&collection_type=expert&sort=popularity

    Note the include_collections=true query param. This is required to have Playlists in the search results.

By default the Search API provides results sorted by "relevance". Relevance is a search specific term, meaning that the search engine judges all matches to the search query and rates them with a floating point number, starting at 0 (zero). There is not maximum relevance score. The Search API provides results in descending order (e.g. order=desc) by default, meaning that you should receive the most relevant result first.

Note -Both of the sort and order parameters are meant to be provided once per Search API call. Providing them more than once may have unexpected results.

Excluding (Filter) - Use the exclude_fields parameter to tell the Search API which fields you are not interested in.

$ curl https://learning.oreilly.com/api/v2/search/?query=python&exclude_fields=description&exclude_fields=publishers

exclude_fields parameter can be anything, with the exception of the id field, which will always be returned.

The full list of result fields names that can be excluded is: academic_excluded, archive, archive_id, authors, date_added, has_assessment, issued, minutes_required, natural_key, timestamp, title, chapter_title, url, virtual_pages, web_url, popularity, description, format, content_format, cover_url, isbn, duration_seconds, content_type, publishers, source, number_of_reviews, average_rating, report_score, events, language, ourn, topics, topics_payload, last_modified_time, video_classifications.
