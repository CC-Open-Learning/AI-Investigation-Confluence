AI Investigation : Data normalization for Instructional plan and Course outline

Created by Jongeon, last modified on Dec 10, 2024

This semester, we have decided to improve the functionality of our chatbot by training it with additional user data. The data to be used for training includes the instructional plan and course outline, and we have chosen to convert this data into JSON format for effective learning.

Why Use JSON for Training

JSON (JavaScript Object Notation) is a format well-suited for structuring and storing data, consisting of a dictionary structure with key-value pairs. It offers several advantages:

  1. Readability and Compatibility
    JSON is easy for humans to read and understand while being compatible with most programming languages, making data processing and utilization more accessible.

  2. Efficient Data Handling
    JSON’s structured format allows for efficient searching and extraction of relevant data, enabling the chatbot to provide accurate information promptly.

  3. Flexibility and Scalability
    JSON format can be easily extended by adding new key-value pairs or modifying existing data, making it ideal for updating training data as needed.

By adopting the JSON format, we can systematically perform data training, storage, and utilization.

Importance of Key-Value Settings

In JSON, keys and values are critical elements that determine data retrieval and training performance. Keys should represent unique identifiers for the data, while values should contain specific and clear information associated with the keys.
For instance, using ambiguous or duplicate keys can confuse the chatbot about which value to return, potentially degrading its performance.

Key Principles of Data Normalization

Data normalization involves structuring data to reduce redundancy and maintain consistency within a database. This process is vital for maximizing chatbot performance. The key principles include:

  1. Elimination of Redundancy
    Ensure the same information is not stored in multiple places to improve data retrieval speed and reduce the possibility of errors.

  2. Structured Key Settings
    Use specific and intuitive keys to clarify the context of the data. For example, “Assignment Due Date” is more informative than just “Due Date.”

  3. Consistency in Values
    Values should be consistently formatted to make them easily interpretable by the chatbot. For instance, dates should follow the same format (e.g., YYYY-MM-DD), and text should be grammatically correct.

  4. Removal of Unnecessary Data
    Exclude data that is not relevant for training to enhance both chatbot performance and data efficiency.
