AI Investigation : Data Handling

Created by Arsh Chauhan, last modified on May 30, 2024

Data Sources-

Data Handling is one of the crucial aspect of Co-pilot studio as it can enable the users to integrate their Data with the AI model so that AI model can interact with their data .

There are other sources to which provides support to Co-pilot. Which includes:

  1. Microsoft 365 - CoPilot can access the data from other Microsoft 365 apps such as Excel, PowerPoint, Outlook and Teams and More. CoPilot pro being a new AI model which is based on GPT 4 is more advanced and it can make changes in that as it can add a entire paragraph based on a prompt and it can also make a entire presentation with only prompt.

  2. Public Websites - Co-pilot studio is able to access the public data which is available on internet and it can give all the necessary information on that.

  3. Local Files - According to the resources Copilot studio is able to access the local files which includes all the files stores on User’s device

Data Handling Practices:

  1. Data Retention - Copilot for Service data is stored in Dataverse(It lets us to store data within set of tables) in addition to the connected CRM Data retention policies differ from other Microsoft 365 applications and non-Dynamics 365 CRM solutions. For example, when your Microsoft 365 subscription ends, your data is retained for 90 days before it’s automatically deleted. However, if you use Copilot for Service, that data isn’t automatically deleted after 90 days when your subscription ends. [1].

  2. Data Storage - When Copilot for Service is connected to Dynamics 365, Copilot for Service data is stored in the corresponding Dataverse instance [1] When Copilot for Service is connected to a non-Dynamics 365 CRM, a default Dataverse instance specific to Copilot for Service is provided to your tenant admin center

  3. Data Deletion - If you need to delete Copilot for Service data (for example, delete data for a specific user), you can choose to manually delete it [1]

  4. Personal Data - Personal data requests in Copilot for Service and Microsoft Copilot Studio are handled in the same way [1]

  5. Security Measures - The data sources linked to the copilot adhere to stringent security measures and controls, managed through the unified admin center of Copilot Studio.[2] This includes data loss prevention, robust authentication protocols, and more

  6. Data Integration - With Copilot connectors in Copilot Studio, organizations can easily expand their copilot knowledge and actions with their business data.[3] Copilot connectors allow organizations to bring together data from Dynamics 365, Microsoft Power Platform, SharePoint, OneDrive, Microsoft Fabric OneLake, public websites, and many other systems of record.

  7. Data Permissions - Copilot adheres to existing data permissions and policies, and its responses to you’re based only on data that you personally can access[4]
