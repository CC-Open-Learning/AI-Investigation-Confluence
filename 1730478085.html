AI Investigation : Research: LLMs' Access to Dynamic Calendars like D2L

Created by Eunie on Apr 03, 2025

LLMs + Web Integration Approaches

LLMs cannot scrape websites like traditional crawlers but can access web content through the following methods:

  1. Function Calling + Backend Scraping

  • Examples: OpenAI Function Calling, LangChain Tools

  • Description: The LLM generates a function call, which is executed in the backend using tools like requests and BeautifulSoup.

{
  "function": "scrape_schedule",
  "arguments": {
    "url": "https://example.com/schedule"
  }
}

  1. RPA Integration (Browser Automation)

  • Tools: AutoGPT, AgentGPT, LangGraph, Open Interpreter

  • Description: Automates browser actions such as logging in, clicking, typing, opening a calendar, taking screenshots, or extracting text.

  1. Browser Tool / Web Browsing Plugin

  • Examples: ChatGPT-4 with Browsing, Bing Copilot

  • Capabilities: Searches the web, visits websites, summarizes visible text content

  • Limitations: Cannot extract deeply embedded or JavaScript-rendered data

The Nature of D2L Calendars

Calendars on D2L (Desire2Learn) are dynamic web pages. The calendar data is loaded asynchronously using JavaScript, meaning:

  • The raw HTML does not contain the actual schedule.

  • Content is rendered on the client side after page load.

Solutions for Dynamic Content (e.g., D2L Calendars)

  1. Use Selenium or Playwright

Automates a browser to load JavaScript-rendered content and extract the final HTML.

from selenium import webdriver

driver = webdriver.Chrome()
driver.get("https://conestoga.desire2learn.com/d2l/le/calendar/1343106")
html = driver.page_source

  1. Check for a Public API

If D2L offers a REST API, it’s the most reliable and stable method to retrieve calendar data.

  1. Manual Copy & Paste

As a fallback, copy content manually from the browser and provide it to the LLM for summarization.

Conclusion

For platforms like D2L, where content is rendered dynamically using JavaScript, basic HTML scraping is not sufficient.

You need either browser automation tools or access to an official API to extract the data effectively.
