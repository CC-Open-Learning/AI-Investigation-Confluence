1. Introduction

Project Overview

The aim of this project is to develop a Digital Teaching Assistant (DTA) chatbot for the college, providing automated, AI-driven support to students. The project leverages advanced AI technologies to enhance the educational experience.

Goals

  • Develop a chatbot that can answer both transactional and abstract questions.

  • Implement up to Level 2 of the DTA Cognitive Autonomy Scale.

→ Refer this Article by Sean Yo - https://medium.com/@sean.yo/evolving-education-with-ai-the-digital-teaching-assistant-and-its-progression-scale-13b604b063d8

2. Specifications

DTA Scale

Level 2 — Interpretive DTA

  • Synthesizes course content(Video Lectures of the course) to answer abstract questions.

  • Data Access: Course content(transcriptions.json).

  • Examples: “Can you explain the main themes of the novel we are studying?”, “How does Newton’s Third Law apply to this physics problem?”, “Can you summarize the key points from the last lecture on climate change?”, “How does the theory of relativity relate to what we learned about quantum mechanics?”

3. Technical Implementations

  • There are 2 approaches to make it possible:

    • Fine-tuning (We tried but due to lack of resources & labelled data, we stopped.)

    • RAG (So, We have used this approach)

Architecture

  • System Architecture Diagram:

  • Description:

→ Embedding:

  • First, we have converted video into “transcriptions.json“.

  • Using Embedding model(Ollama Embeddings' mxbai-embed-large), converted “transcriptions.json” into small Documents/chunks and then into vectors(array of numbers).

  • Stored all vectors into Vector Database(ChromaDB).

→ Retriever:

  • Retriever - Which fetches related documents from Vector DB based on the prompt/user question of the course.

  • Those retrieved documents will be sent to LLM(Groq’s mixtral-8x7b) along with user prompt and system prompt.

    • Other Open source LLMs - Llama3, Gemini, etc. one can use.

  • The response that generated by the LLM is our User’s answer.

Technologies Used

  • LangChain: For managing the workflow of the AI models.

  • Chroma Vector Database: For storing and retrieving vectorized data.

  • RAG (Retrieval-Augmented Generation): For generating responses based on retrieved data.

  • Mistral AI: As the primary language model.

  • LLaMA3: For additional AI capabilities.

  • Groq API: For accessing Open Source LLMs using API Calls.

  • Ollama Embeddings: Local stored model for creating vector representations of text data.

5. Concepts

  • Here’s a list of key concepts with brief descriptions for creating a RAG (Retrieval-Augmented Generation) based chatbot:

1. RAG (Retrieval-Augmented Generation):

  • Description: A hybrid model that combines retrieval-based and generative approaches. It retrieves relevant documents or data points from a knowledge base and uses them to generate more accurate and contextually relevant responses.

2. Retriever:

  • Description: A component that searches through a database or document store to find relevant information or data points that can assist in generating a response. Common retrieval methods include BM25 and vector similarity search.

3. Generator:

  • Description: A language model that generates responses based on the retrieved information. It ensures that the generated response is coherent and contextually appropriate.

4. Vector Embeddings:

  • Description: Numerical representations of text that capture semantic meaning. These embeddings are used for similarity searches, where queries and documents are compared based on their vector representations.

5. Chroma Vector Database:

  • Description: A specialized database designed to store and retrieve vector embeddings efficiently. It is used to manage the large sets of embeddings required for fast and accurate retrieval.

6. LangChain:

  • Description: A framework for developing language model applications, enabling the seamless integration of multiple language models and tools, such as retrievers, to create a coherent chatbot experience.

7. LLaMA3:

  • Description: A language model that can be used in the generator component of a RAG system, known for its efficiency and ability to generate human-like text.

8. Mistral AI:

  • Description: Another language model option, often used for its balance of performance and resource efficiency in generating responses.

9. Ollama Embeddings:

  • Description: A type of vector embedding specifically used to enhance the performance of the retriever by improving the accuracy of similarity searches.

10. Groq API:

  • Description: An API that provides access to high-performance AI hardware, enabling faster computation and model inference for the chatbot.

11. Prompt Engineering:

  • Description: The process of designing prompts that effectively guide the language model in generating desired responses, ensuring that the output is relevant and accurate.

12. Knowledge Base:

  • Description: A structured database or document store that contains the information the retriever searches through. It can include text documents, FAQs, or any structured data.

13. Fine-tuning:

  • Description: The process of training a language model on a specific dataset to improve its performance on domain-specific tasks or responses.

14. Contextual Retrieval:

  • Description: An advanced retrieval technique that not only considers the current query but also the conversation’s history to provide more contextually relevant documents.

These concepts form the foundation of a RAG-based chatbot, helping integrate retrieval with generative AI to create more intelligent and responsive conversational agents.
