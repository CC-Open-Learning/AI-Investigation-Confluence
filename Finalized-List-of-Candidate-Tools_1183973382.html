AI Investigation : Finalized List of Candidate Tools

Created by Diego Andres Bolanos Osejo (Unlicensed), last modified on Jun 21, 2024

Required Tools: - Azure AI + Machine Learning services

  1. GPT-2 (OpenAI)

Rationale:

  1. Ease of Use: Well-supported and easy to deploy with frameworks like Hugging Face.

  2. Resource Efficiency: Requires lower computational resources compared to newer, larger models.

  3. General-Purpose Performance: Capable of handling a wide range of tasks including text generation and basic conversational AI.

  4. Rapid Deployment: Suitable for quick setups and cost-effective solutions, making it ideal for educational purposes.

Use Cases: General educational chatbots, basic enterprise applications, rapid prototyping.

  1. GPT-Neo (EleutherAI)

Rationale:

  1. Scalability: Available in various sizes (1.3B, 2.7B, 6B parameters), allowing flexibility based on resource availability and performance needs.

  2. Open-Source: Fully open-source, providing flexibility in usage and modification.

  3. Comparable Performance: Offers performance that is competitive with larger proprietary models at no cost.

  4. Community and Support: Backed by EleutherAI and a strong community, ensuring ongoing improvements and support.

Use Cases: Intermediate educational chatbots, mid-scale enterprise applications, projects requiring open-source flexibility.

  1. BERT (Google)

Rationale:

  1. Strong Contextual Understanding: Excels in tasks requiring deep comprehension of text, such as question answering and text classification.

  2. Versatile Applications: Can be fine-tuned for various NLP tasks, making it highly adaptable for different chatbot functionalities.

  3. Efficient Deployment: Generally, more efficient in terms of inference compared to generative models, especially for understanding and processing input.

Use Cases: FAQ bots, educational content comprehension, structured data extraction.

 

  1. LLaMA 3 (Meta AI)

Rationale:

  1. State-of-the-Art: Represents the latest advancements in LLMs, offering high performance and efficiency.

  2. Highly Customizable: Can be fine-tuned extensively to meet specific requirements, making it suitable for specialized applications.

  3. Efficient and Scalable: Designed to be resource-efficient while delivering competitive performance.

Use Cases: Highly specialized educational tools, customized enterprise applications, projects requiring cutting-edge performance.

  1. Falcon (Technology Innovation Institute)

Rationale:

  1. Balanced Performance: Offers robust performance with a reasonable balance of resource demands.

  2. Versatile and Adaptable: Well-suited for a range of applications from general conversational AI to more specialized tasks.

  3. Strong Open-Source Alternative: Competes with proprietary models while being freely available, supporting customization and innovation.

Use Cases: Versatile educational chatbots, adaptable enterprise applications, mid-to-high complexity tasks.

 

Recommendations

  1. General Educational or Enterprise Chatbots:

    • GPT-2 and GPT-Neo (1.3B or 2.7B): These models are ideal for general-purpose chatbots due to their ease of use and resource efficiency.

  2. Intermediate to Advanced Chatbots:

    • LLaMA 3: Suitable for more demanding applications that require high performance and sophisticated language capabilities.

  3. Specialized or Contextual Understanding Tasks:

    • BERT: Best for applications needing deep text comprehension, such as answering detailed questions or extracting structured information.

  4. Versatile and Adaptable Applications:

    • Falcon: Offers a good balance for both general and specialized use cases, providing strong performance with manageable resource demands.

Decision Criteria

  1. Ease of Development and Integration:

    • Streamlit provides rapid development and easy integration with Python-based AI libraries, making it suitable for quick prototyping and deployment.

    • Angular and NLUX require more setup and integration effort, particularly with front-end frameworks (Angular or React.js) and backend services.

  2. Scalability and Flexibility:

    • Angular Tech Stack and NLUX Tech Stack offer robust solutions for large-scale applications with Docker, Kubernetes, and cloud providers. They are highly scalable and flexible for enterprise-grade applications.

    • Streamlit is more suited for smaller-scale applications or prototypes, with limited scalability in its community cloud deployment.

  3. Specific Requirements:

    • Streamlit: Best for rapid prototyping and development with a focus on AI/ML applications using Python.

    • Angular: Suitable for more complex, enterprise-grade applications requiring a robust front-end framework.

    • NLUX: Ideal for applications leveraging React.js for a highly interactive user experience.

Recommendation

Considering the need to build an end-to-end AI chatbot that integrates with external APIs like Brightspace for large educational content processing, the Angular Tech Stack is recommended. This stack provides a balanced approach to scalability, integration with AI models, and a robust frontend framework for user interaction. It is suitable for enterprise-level applications and can handle large datasets effectively.
