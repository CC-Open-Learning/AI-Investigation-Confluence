AI Investigation : Copilot Security Concerns

Created by Josslyne Shantz (Unlicensed), last modified on Jun 06, 2024

One security concern for Copilots is improper permissions. Copilot uses permissions set in Microsoft 365. Excessive broad access can lead to the dissemination of sensitive data.

Another security concern is inaccurate data classification. If a mistake is made when labeling documents, the mislabeled documents could be shown when they shouldn’t be or not be shown when they should be.

Another security concern is Copilot-generated content as it doesn’t automatically apply sensitivity labels when replying to a prompt.

One more security concern is jailbreaking techniques. If a jailbreaking technique has not been patched for GPT-4 or other LLMs that Copilot uses, they can be used in Copilot, leading to it answering prompts it shouldn’t. There is a possibility that we may be able to use Copilot’s Prompt Shield and or Moderation APIs which are part of Azure AI Content Safety. Currently, only Canada East is supported for the Moderation APIs, not the Prompt Shield preview. We are in Canada Central, so we have access to Azure Content Moderator which is deprecated as it’s been replaced with Content Safety. Content Moderator will be retired by February 2027.
