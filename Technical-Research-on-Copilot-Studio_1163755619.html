AI Investigation : Technical Research on Copilot Studio

Created by Diego Andres Bolanos Osejo (Unlicensed), last modified on Jun 12, 2024

Introduction:

  • Copilot Studio: Provides a complete experience for creating, managing, and publishing copilots.

  • Capabilities: Allows building sophisticated and secure copilots, connecting to enterprise data, and integrating various backend systems and APIs.

Technical Architecture for Microsoft Copilot Studio

  1. Conversation Volumes:

    • No current limitations on monthly message volume or growth (depending on the license).

    • Expected increase in chat sessions during admissions periods.

    • Peak concurrent chats depend on the number of users at the start of the academic year.

  2. Performance:

    • Initial chat load time: 5 seconds.

    • Maximum latency for responses: 3 seconds.

    • Long-running actions: Display a waiting message and typing indicator.

  3. Technical Challenges:

    • Connecting to on-premises resources not exposed to the internet.

    • Allowing users to download conversation transcripts.

    • Ensuring data protection and user privacy during testing.

Natural Language Understanding

  • NLU Basics: Specializes in comprehending human language, focusing on context, sentiment, and intent.

  • Configuration: Trigger phrases and custom entities.

  • In Copilot Studio, topic or action triggering can be done in different ways:

    • Azure CLU (Conversational Language Understanding) – the modern equivalent of LUIS.

      • Allows to further customize the intent triggering model for better intent recognition or to address specific industry requirements.

    • GPT LLM-based

      • Can handle multiple intents and chain topics and/or plugins.

      • Automatically generate questions for missing inputs and answers complex entities  and questions from the conversation context.

      • Configuration is done by simply describing topics, plugin actions, and inputs/outputs.

  • Azure CLU is a feature of Azure AI Language.

Intent recognition and slot filling

Defining topics trigger phrases: From an existing FAQ base and from agent chat transcripts

  • Trigger phrases train your copilot's natural language understanding (NLU) model.

  • It’s important to appropriately configure them to make sure that the right topic triggers for the user utterances.

  • Handling uncertainty by suggesting potential topics or using fallback.

Use of entities: There are two key concepts: Entity extraction and slot filling

  • Slot filling relates to the use of entities that let a copilot acquire and use information more efficiently by identifying and extracting them from the user query.

  • For example, when a user asks:
    “I’d like to order 3 large blue t-shirts”
    Natural Language Understanding (NLU) can immediately understand:

    • Topic is Order.

    • Quantity is 3.

    • Color is Blue.

    • Size is Large.

    • Item Type is T-Shirt.

  • And the related questions be skipped.

Test topic triggering with test utterances:

  • Bulk testing to make sure the relevant topics are triggered based on a set of test user utterances.

Note: Refer to User Interactions and Dialogues for more details

Azure CLU Integration

  • Uses: Supports additional languages, industry-specific dictionaries, and complex entity extraction.

  • Custom entity extractions: To do silent or lucky slot filling without having to ask questions and to handle unsupported scenarios in Copilot Studio such as source and destination city in the same phrase.

  • Service Quotas: Setup using the S tier, with fewer than 1,000 calls per minute expected.

    • You can integrate a Conversational Language Understanding (CLU) model with a Copilot Studio copilot

    • Mapping Azure CLU intents with Copilot Studio topics.

    • Copilot Studio prebuilt entities can be used in conjunctions with Azure CLU entities.

    • Using Azure CLU requires separate Azure resources and involves separate costs.

Topic structure example for disambiguation

Topic structure

Design: Custom topics with relevant trigger phrases and a parent-child structure. Unrecognized intents will trigger generative answers and fallback.

Disambiguation Topics: Clarifying questions for operations and systems. (e.g. create, unlock, suspend, etc.) and system (e.g. SAP, ServiceNow, Microsoft, etc.)

Reusable Topics: For repeated dialog paths.

  • Topics are discrete conversation paths that, when used together, allow for users to have a conversation with a copilot that feels natural and flows appropriately.

  • While there's no one size fits all, given how topics can be triggered, it's a good practice to distinguish between:

  • Topics that will trigger based on user utterances. If you have trigger phrases that overlap multiple topics, consider having a catch-all topic and then redirect to other topics after clarifying questions. With entity extraction and slot filling, clarifying questions can be skipped if already answered.

  • Topics that will trigger when called from a redirect action, activity or event. These can be called by multiple topics and can have input and output variables. They're ideally reusable, bite-size, topics.

  • A topic can also be both, triggered through intent recognition or by an explicit redirect.

  • Conversational Boosting and Fallback: Triggering topics when no matching topic is found.

Handling unrecognized intents

  • Fallback Management: Using corporate resources and fallback to a ChatGPT experience if needed.

  • Fallback Rate: Initially 25%, with regular reviews to enhance topics.

Implementing AI Capabilities to the custom Chatbot

Generative answers

  • Data Sources: Public websites, SharePoint sites, and ServiceNow knowledge bases.

  • Answer Format: Brief and friendly responses (40 words max).

  • Integration: Generative answers used in custom topics and conversational boosting.

Mostly in custom topics that will contain trigger phrases and entity questions that will direct the user towards the right data source. Conversational boosting will use public FAQ websites

Advanced Scenarios and Dynamic Chaining

As a last resort and to handle chitchat and unrelated queries, the copilot will be able to answer in a ChatGPT manner (e.g. writing a SQL query, etc.)

  • Use of plugin actions: Performing system actions and handling missing properties and letting AI ask for any missing required property for the configured API

  • Dynamic Chaining: Replacing traditional NLU with GPT-based models for advanced multi-intent detection (for topics and plugin actions) and entity extraction (even for entities of the same type).

Generative builder (copilot)

  • Ease of Use: Helps business users get started without an empty topic canvas.

  • Adaptive Cards: Summarize information gathered from multiple questions.

Other AI Requirements:

  • Sentiment Analysis: Escalating to live agents if the tone is negative.

  • Transcript Analysis: Suggesting new topics or trigger phrases.

  • AI Builder Integration: For image recognition and text extraction.

Integrations

  • Power Automate Cloud Flows: No-code/low-code integration, clear separation of logic, and monitoring capabilities.

  • HTTP Requests and Connectors: Synchronous, no-code/low-code, with parsing and error handling.

  • Bot Framework Skills: Synchronous execution and support for private endpoints.

Power Automate and HTTP/Connectors

  • Performance: Ensuring quick execution of cloud flows and HTTP requests.

  • Custom Connectors: Power Automate’s extensive connector capabilities.

  • Execution Time Limit: Cloud flows must return within 100 seconds to avoid timeout.

Glossary of Copilot Studio Concepts

Copilot Studio concept

Description

Topics

Use topics to organize conversation flow or paths. A topic has a set of trigger phrases that indicate when the copilot should start the topic and a set of nodes that describe the topic's conversation path.

Trigger phrases

Add trigger phrases to a topic for the phrases, keywords, and questions that a customer is likely to type related to a specific issue. Copilot uses natural language understanding to parse what a customer types and find the most appropriate topic.

Nodes

Use nodes, such as messages, questions, and conditional branches, on the authoring canvas to create a topic's conversation path.

Response variations and question variations

Use response and question variations to add variety to your copilot's messages and questions.

Quick replies

Use quick replies to provide default reply options to the customer.

Entities

Define and use entities to extract semantic information from what a customer types.

Variables

Use variables to track state.

Power Fx

Use Power Fx to create expressions.
