Workflow for Creating and Deploying the Chatbot

Phase 1: Planning and Data Collection

  1. Define Objectives:

    • Determine the main purposes of the chatbot (e.g., answering course-related questions, providing grade information, notifying about assignment due dates).

  2. Identify Data Sources:

    • Course Content: Syllabus, lecture notes, textbooks.

    • Assignment Data: Assignment descriptions, due dates.

    • Grades Data: Student grades, feedback from instructors.

  3. Gather and Organize Data:

    • Ensure all data is up-to-date, accurate, and in a structured format suitable for processing.

Phase 2: Setting Up the Environment

  1. Create an Azure Account (if not already available):

    • Sign up for an Azure account to access Microsoft Copilot Studio and other necessary services.

  2. Access Microsoft Copilot Studio:

    • Log in to Copilot Studio through your Azure account.

  3. Create a New Project:

    • Start a new project dedicated to your chatbot.

Phase 3: Data Preparation and Integration

  1. Upload Datasets:

    • Import course content, assignment data, and grades data into Copilot Studio.

  2. Preprocess Data:

    • Clean and structure data to ensure it is suitable for training the AI model.

Phase 4: Chatbot Design and Development

  1. Define Intents and Entities:

    • Identify key intents (e.g., explaining laws of motion, checking grades, querying assignment deadlines).

    • Define entities related to these intents (e.g., specific laws, grade categories, assignment names).

  2. Train the Natural Language Understanding (NLU) Model:

    • Provide examples of user queries for each intent to train the NLU model.

  3. Create Conversational Flows:

    • Design the dialogue structure for each type of query.

    • Use Copilot Studio's tools to build and refine these conversational flows.

  4. Implement User Authentication:

    • Integrate authentication mechanisms to securely fetch and display user-specific data like grades.

Phase 5: Integration with eConestoga (D2L)

  1. API Integration:

    • Use D2L Brightspace APIs to dynamically access and update data within the eConestoga portal.

    • Set up secure API calls, handling OAuth tokens as required.

  2. Embed the Chatbot in eConestoga:

    • Use iframes or embedding options provided by D2L to integrate the chatbot interface into the portal.

    • Ensure the chatbot is easily accessible to users.

Phase 6: Testing

  1. Functional Testing:

    • Test each intent and conversational flow to ensure they work correctly.

    • Verify that the chatbot can accurately retrieve and display grades, assignment due dates, and course content.

  2. User Testing:

    • Conduct user testing to gather feedback on the chatbot's performance and usability.

    • Make necessary adjustments based on user feedback.

Phase 7: Deployment

  1. Deploy the Chatbot:

    • Finalize and deploy the chatbot within the eConestoga portal.

    • Ensure all integrations and data connections are active and functioning correctly.

  2. Launch:

    • Announce the availability of the chatbot to users.

    • Provide guidance on how to interact with the chatbot.

Phase 8: Monitoring and Maintenance

  1. Monitor Performance:

    • Use analytics tools to track chatbot usage and performance.

    • Monitor user interactions to identify areas for improvement.

  2. Regular Updates:

    • Update datasets regularly to keep information current.

    • Continuously refine the NLU model based on new data and user feedback.

  3. Ongoing Maintenance:

    • Address any technical issues promptly.

    • Enhance the chatbot's capabilities over time to better serve user needs.
