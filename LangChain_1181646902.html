AI Investigation : LangChain

Created by Jongeon, last modified by Paras Rupani on Sept 16, 2024

LangChain is a framework for developing applications that leverage language models. It goes beyond simply calling language model via an API, supporting the development of more powerful and differentiated applications through various functionalities and combinations.

Why do we use LangChain?

Let`s talk about assuming the use of ChatGPT. ChatGPT cannot represent an LLM, but most LLMs suffer from similar limitations.

Limitations of ChatGPT

  1. Information access restriction

    1. Since ChatGPT (GPT-3.5) is an LLM (large language model) trained on data up to 2021, it cannot respond to information from 2022 onwards, or it may provide false answers.

  2. Token limitation

    1. The models provided by ChatGPT, GPT-3.5 and GPT-4, have input token limits of 4096 and 8192 tokens, respectively.

  3. Hallucination

    1. When asked factual questions, it often gives irrelevant or false answers.

How can we overcome the limitations?

  1. Prompt engineering: Optimize input prompts to improve the quality of model-generated outputs, gaining significant attention in recent advancements.

  2. Multi-task Learning: Train models to handle multiple tasks simultaneously, enabling a single model to perform various tasks by integrating diverse datasets and task types to enhance performance.

  3. Fine-tunning: Update the existing deep learning model`s weights to tailor it for specific use cases. For example, Bloomberg GPT demonstrates this approach in financial news analysis.

  4. In-context Learning: Provide context so that the model can generate output based on that context, enhancing relevance and accuracy.

  5. Transfer Learning: Utilize models pretrained on different tasks to enhance performance in new applications, closely linked with fine-tuning for optimal adaptation.

  6. N-shot Learning: Adjust the deep learning output by providing 0 to n examples of output for the intended use case, addressing data scarcity issues effectively.

Pros

Cons

Prompt Engineering

Enhances model output quality and accuracy through optimized input prompts.

Requires design and adjustment of input prompts, may involve human intervention.

Multi-task Learning

Increases resource efficiency by handling multiple tasks with a single model.

Risk of task interference and requires complex training and model design.

Fine-tuning

Enables model optimization for specific tasks with minimal data, achieving high performance.

Requires additional data and computational resources, prone to overfitting.

In-context Learning

Generates accurate outputs by considering provided context.

Output quality varies based on input context quality, requires complex context provision.

Transfer Learning

Enhances performance in new tasks using pretrained models.

Potential issues with unsuitable pretrained models and domain adaptation.

N-shot Learning

Improves performance with limited data, addressing data scarcity.

Performance may degrade with insufficient examples, influenced by data quality and diversity.

Summary

LangChain serves as a platform that adjust language models in various ways and enables them to be applied to specific tasks. It functions as a tool supporting diverse learning and application techniques.

Appendix B

Step-by-step basic workflow.

  • Step 1: Load the document

  • Step 2: Splitting the document into chunks

  • Step 3: Use Embedding against Chunk Data and convert to vectors

  • Step 4: Save data to the Vector database

  • Step 5: Take data(question) from the user and get the embedding

  • Step 6: Connect to VectorDB and do a semantic search

  • Step 7: Retrieve relevant responses based on user queries and send them to LLM

  • Step 8: Get an answer from LLM and send it back to the user

Appendix C

Embedding: Representing words or sentence in a vector space to preserve meaning of relationships for computer understanding.

Corpus: Collection of text data used for analysis in natural language processing.

Vector store: Repositories for storing and managing word or sentence embeddings efficiently.
