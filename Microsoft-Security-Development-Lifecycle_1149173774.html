AI Investigation : Microsoft Security Development Lifecycle

Created by Josslyne Shantz (Unlicensed) on Jun 03, 2024

The Microsoft Security Development Lifecycle is a set of practices and tools to help developers build more secure software. There are 10 key security practices that will help integrate security into each stage of the developers over development process.

Security risks and the need to mitigate them can occur at any time in the development lifecycle. Here are some ways you can help mitigate risks before they appear:

  • During the design phase, ensure that the design doesn’t naturally allow attackers to easily gain unauthorized access to the workload, data, or other business assets.

  • While coding, ensure that the code doesn’t allow attackers to easily take control of the application so as to perform unauthorized actions that could cause harm to customers, data, employees, systems, or other business assets. Developers should also work in a secure environment that doesn’t allow attacker to do such things without the developers knowledge.

  • While building and deploying the product, ensure that the continuous integration and continuous deployment doesn’t allow unauthorized users to alter code and allow attackers compromise it.

  • When running the application, ensure the environment used to run the code follows best security practices across everything to prevent attacker from compromising and abusing the work load. This includes the adoption of well-established best practices, security baseline configurations, and more.

All of the afore mentioned stages (design, coding, building and deploying, running) should follow Zero Trust principals; assuming there will be a breach (assume being compromised), explicitly verifying trust, granting the lowest level of privilege that a user requires, machine/service identity, and application component.

Ten Key Security Practices

  1. Establish security standards, metrics, and governance

  2. Require use of proven security features, languages, and framework

  3. Perform security design review and threat modeling

  4. Define and use cryptography standards

  5. Secure the software supply chain

  6. Secure the engineering environment

  7. Perform security testing

  8. Ensure operational platform security

  9. Implement security monitoring and response

  10. Provide security training
