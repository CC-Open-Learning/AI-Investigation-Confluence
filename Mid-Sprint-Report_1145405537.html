Microsoft Copilot

Microsoft Copilot integrates AI capabilities into various Microsoft applications to enhance productivity, safeguard businesses, and extend AI functionalities. Key uses include:

  • Web Browsing: Integrated with Microsoft Edge and Bing for content creation and research.

  • Productivity Apps: Embedded in Microsoft 365 (Word, PowerPoint, Outlook) to assist with document generation, presentations, and email management.

  • Business Processes: Integrated into Dynamics 365 for customer service, sales, and supply chain optimization.

  • Data Analytics: Supports data analysis in Microsoft Fabric and Power BI.

  • IT and Security: Aids in managing IT infrastructure and security via Azure and Microsoft Copilot for Security.

  • Software Development: GitHub Copilot helps developers with code generation and documentation.


Copilot Studio

Copilot Studio is designed to work well for low-code development scenarios in which technically proficient business users or developers can create conversational AI experiences. The resulting copilot is a fully managed SaaS (software as a service) solution hosted in your Microsoft 365 environment and delivered through chat channels like Microsoft Teams. With Copilot Studio, the infrastructure considerations and model deployment details are taken care of for you, making it easy to focus on creating an effective solution.

Features and Advantages of Studio

  1. Comprehensive Capabilities

    1. Creation: Copilot Studio provides tools to build sophisticated copilots with ease.

    2. Management: Centralized dashboard for managing all copilots.

    3. Publishing: Streamlined process to publish copilots across various platforms.

  2. Integration and Connectivity

    1. Enterprise Data: Connect to enterprise data sources and integrate various backend systems and APIs.

    2. Natural Language Understanding: Leverages Azure CLU and GPT models for advanced NLU, ensuring accurate intent recognition and slot filling.

  3. Enhanced Performance

    1. Scalability: No current limitations on message volume, supporting high traffic during peak periods.

    2. Latency: Optimized for low response times, ensuring a smooth user experience.

  4. Customization and Flexibility

    1. Trigger Phrases and Custom Entities: Define specific triggers and entities tailored to business needs.

    2. Dynamic Chaining: Replace traditional NLU with GPT-based models for advanced multi-intent detection.

  5. Advanced AI Features

    1. Sentiment Analysis: Automatically escalate to live agents if negative sentiment is detected.

    2. Generative Answers: Provides brief and friendly responses sourced from public websites, SharePoint, and ServiceNow knowledge bases.

Architecture of copilot studio

  • Plugins: The core feature of Co-Pilot Studio, which are reusable pieces of code designed to perform specific tasks within a co-pilot. 

  • Plugin Authoring Section: A user-friendly GUI where users can create and edit plugins. 

  • Plugin Registry: A storage system for plugins that allows authorized users to access and use them across multiple co-pilots. 

  • Creator Layer: The layer where users create or edit plugins, with four main types: 

    • Prompt Plugin: Utilizes NLP and NLG to answer user queries. 

    • Flow Plugin: Executes multi-step workflows using Power Automate. 

    • Connector Plugin: Connects to external systems or data sources like GitHub or Salesforce.

    • Topic Plugin: Addresses specific user utterances or queries. 

  • Admin Layer: Uses the Microsoft Admin Center to manage app installations and plugin discovery and execution. 

  • Data/AI Layer: Manages plugin metadata and execution information, facilitating discovery at tenant and user levels. 

  • Runtime Execution Layer: Executes the plugins, including interactions with APIs and external systems. 

  • Bot Framework: Provides concepts for composers/users to organize conversation flows and paths using topics, intents, actions, prompts, and entities. 

  • Fallback Option: Allows the co-pilot to use AI general knowledge to respond to customer questions when search results are missing or permissions are insufficient.

Security and Implementation plans

  • Data Loss Prevention (DLP): Power Platform includes robust DLP features to secure data. It’s essential to configure DLP policies for copilots effectively. 

  • Governance and Security: 

    • Publishing Control: Admins can disable the ability to publish copilots with generative answers and actions. 

    • Organization-Wide Disable Option: Copilot can be disabled for the entire organization through a support request. 

    • Geographic Data Movement: Restrict data movement for Copilot Studio’s generative AI features outside the United States. 

    • Conversational Plugins: Enablement of Copilot Studio conversational plugins is supported. 

  • DLP for Co-pilots: 

    • Automation and Data Sources: Co-pilots can be rapidly built and connected to various data sources, including external and third-party services. 

    • Exposure Risks: There’s a potential risk of data leakage or unauthorized access due to connections with external services. 

  • DLP Policy Management: 

    • Policy Creation: Admins can create DLP policies in the Power Platform admin center, requiring tenant admin or Environment Admin role. 

    • Connector Governance: New connectors are governed by updating the DLP policy to include them. 

  • Connectors: 

    • Types: There are prebuilt and custom connectors for various APIs and services. 

    • Classification: Connectors are classified into Business, Non-business, and Blocked groups for DLP enforcement. 

  • Important Notes: 

    • Data Group Changes: Admins can change the data group of connectors, affecting accessibility for users. 

    • Blockable Connectors: Most connectors are blockable, except for core Microsoft platform functionalities. 

  • Endpoint Filtering: 

    • Control: Admins can govern specific endpoints that makers can connect to when building apps or chatbots. 

    • Current Availability: Configuration of endpoint filtering with DLP policy is available for six connectors, including HTTP, SQL Server, and Azure Blob Storage. 

  • Copilot Studio Connector Configuration: 

    • Available Connectors: A list of Copilot Studio connectors available in the Power Platform admin center is provided. 

    • DLP Configuration: Instructions on how to block copilot makers from connecting to certain services like Application Insights, Facebook channel, etc.

Technical Limitations of Copilot

  1. On-Premises Resource Connectivity

    1. Challenges in connecting to on-premises resources that are not exposed to the internet.

  2. Data Protection and Privacy

    1. Ensuring user data protection and privacy during testing and real-time usage.

  3. Azure CLU Integration

    1. Custom Entity Extraction: Requires additional configuration and separate Azure resources, incurring additional costs.

    2. Service Quotas: Limited to fewer than 1,000 calls per minute, potentially impacting performance during high traffic periods.

  4. Latency Issues

    1. Initial chat load time of 5 seconds and a maximum response latency of 3 seconds may not meet all real-time interaction needs.

  5. Incorrect Access Control - Inadequate access control can allow unauthorized users to access sensitive data and compromise the security of the system. Organization should be able to determine the appropriate things user is allowed to do.

  6. Fallback and Unrecognized Intents

    1. Fallback Rate: Initially set at 25%, requiring regular reviews and updates to enhance topic recognition.

    2. Fallback Management: Utilizes corporate resources and falls back to a ChatGPT experience if necessary, which might not always align with enterprise requirements.
