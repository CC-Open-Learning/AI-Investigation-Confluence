AI Investigation : Required software for the build phase

Created by Diego Andres Bolanos Osejo (Unlicensed), last modified on Jun 24, 2024

Copilot Studio

AI Integration and Logic:

  • Copilot Studio AI Logic: Utilize the low-code platform to design AI workflows and logic.

  • Azure Cognitive Services: Implement advanced cognitive capabilities for enhanced AI features.

  • Azure OpenAI: Integrate Azure’s OpenAI services for access to powerful language models.

Deployment and Hosting:

  • Copilot Studio Deployment Manager: Seamlessly deploy your Copilot application.

  • Azure App Service: Host your application on Azure for scalability and reliability.

Data Management:

  • Copilot Studio Data Connectors: Connect to Azure data services with ease.

  • Azure Cosmos DB: Manage application data across global distribution.

  • Azure Blob Storage: Store large files and unstructured data efficiently.

Model Training and Fine-Tuning:

  • Copilot Studio ML Workbench: Access in-studio tools for model training.

  • Azure Machine Learning: Gain granular control over model training and management.

Monitoring and Analytics:

  • Copilot Studio Insights: Monitor application performance within the studio.

  • Azure Monitor and Application Insights: Analyze telemetry data for actionable insights.

Collaboration and Version Control:

  • Copilot Studio Collaboration Tools: Collaborate in real-time within the studio.

  • Azure DevOps: Use Azure DevOps for source control and CI/CD pipelines.

Angular

Backend:

  • NestJS: A framework for building efficient, reliable and scalable server-side applications.

  • Node.js: JavaScript runtime built on Chrome’s V8 JavaScript engine.

  • Express.js: Web application framework for Node.js.

AI Services Integration:

  • Azure OpenAI: Services for accessing OpenAI’s language models.

  • Azure Cognitive Service for Language: Cloud-based natural language processing service.

  • LangChain: A library for chaining language model calls to external APIs.

Frontend:

  • Angular: A platform for building mobile and desktop web applications.

  • TailwindCss: A utility-first CSS framework for rapidly building custom designs.

AI Integration:

  • Brightspace API: An API to integrate with the Brightspace learning management system.

Deployment:

  • Docker: A platform for developing, shipping, and running applications.

  • Kubernetes: An open-source system for automating deployment, scaling, and management of containerized applications.

  • Azure DevOps: Services for continuous integration and continuous deployment.

  • Azure App Service: A fully managed platform for building, deploying, and scaling web apps.

Storage:

  • Azure Cosmos DB: A globally distributed, multi-model database service.

  • Azure Database for MySQL: A fully managed database service for app developers.

  • Azure Blob Storage: A service for storing large amounts of unstructured data.

  • Azure File Storage: A service that offers file shares in the cloud.

Data Ingestion and Storage:

  • Azure Databricks: An Apache Spark-based analytics platform optimized for Azure.

Compute Resources for Training:

  • Azure ML Compute Cluster: A managed-compute infrastructure for running machine learning workloads.

  • VMs and GPU Instances (N-Series): Virtual machines with GPU capabilities for AI and deep learning.

Model Selection and Fine-Tuning:

  • Azure Machine Learning (Azure ML): A cloud service for accelerating and managing the ML project lifecycle.

Model Deployment:

  • Azure ML Endpoints: Service for deploying machine learning models as API endpoints.

  • Azure Kubernetes Service (AKS): A service for deploying, managing, and scaling containerized applications using Kubernetes.

  • Azure Functions: A serverless compute service that lets you run event-triggered code without having to explicitly provision or manage infrastructure.

Monitoring and Logging:

  • Azure Monitor: A service that provides detailed performance and utilization data.

  • Azure Application Insights: An extensible Application Performance Management (APM) service for developers.

  • Azure ML Metrics: A feature of Azure ML for tracking and logging model training performance and metrics.

Deployment Plan:

  • Azure Subscription: Required for accessing Azure services.

  • Azure ML Workspace: An environment for managing the Azure ML resources.
