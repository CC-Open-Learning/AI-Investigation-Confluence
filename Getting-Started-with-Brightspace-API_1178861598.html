AI Investigation : Getting Started with Brightspace API

Created by Arsh Chauhan, last modified on Jul 30, 2024

Brightspace API

Technological Features

  • All the Architecture of the D2L Brightspace API is based on the Web2.0 and cloud servers that permit richly integrated apps, easy configuration and operational independence. All the access to the D2L space is supported through HTTP and HTTPS

  • OAuth 2.0 is supported by the API so that it can ensure transparently all authentication scenarios involving applications. They can also ensure that all the API routes into LMS occur within the context of particular user login, service administrators can be confident that application is safe to use

  • Open source library wrappers are also provided for a number of languages with an emphasis on prioritizing the more sophisticated authentication steps of application’s interaction with LMS

Security Features

Brightspace implements LTI Standards to provide third parties with a way to integrate LTI’s service with Brightspace user experience in standard based way incorporating robust, service to service trust. Now the question is what is LTI and what are the advantages of LTI. LTI is Learning Tools Interoperability and they are used to connect learning tools with institution’s learning environment without logging into each tool. LTI supports high level of security for passing data about the users, their institutional enrollment and roles. For more information about LTI refer - https://www.1edtech.org/standards/lti

Beginning Steps

To use the Brightspace API using OAuth we have four entities in that

  1. APIs - Its like application or data APIs. Also known as the resource server in the official specifications

  2. Application - Its the third part application that use the API(Like our chatbot). It is also known as the client.

  3. Authorization Server - It provides access token among other responsibilities

  4. User - A student or instructor who are going to use that application

Register application

By using Brightspace Manage extensibility tool we can register the application with Brightspace OAuth2 authentication service. That application will use its ClientID as a public “name” to uniquely identify itself to the Brightspace Authentication Service later when requesting authentication tokens.

Creating LMS Test account

Every Brightspace API call happens within a calling user context, and the LMS will limit the functionality it extends to your call depending upon the permissions that user would have logged in to the LMS and using through the standard browser interface.{https://docs.valence.desire2learn.com/basic/firstlist.html

Choosing Dev Environment

Brightspace API is a platform neutral for development such as OAuth2 and the use of HTTP transport

Steps to Register for an Application

  1. As a System Administrator, navigate to the Manage Extensibility admin tool

  2. Click the OAuth2.0 Tab.

  3. Click the Register an app button

  1. In the Application Name field, enter the name of your app. Brightspace Learning Environment users see this name on the consent page.

  2. In the Redirect URI field, enter the URI for the page that you want users to be redirected to after authorization is successful.

  3. In the Scope field, enter a value that represents the scope of the resources you want the app to be able to access on behalf of the user. This value must take the form of a space-delimited string that describes the resource group type, the resource, and the permissions you want the app to have (group:resource:permissions group2:resource2:permissions2). For example, data:aggregates:read.

  4. In the Access Token Lifetime field, enter a value in seconds that represents the amount of time that you want the token to be valid for. After this time, the token becomes invalid and the user is prompted to log in again.

    Note that you must provide a value between 1800 seconds (30 minutes) and 72000 seconds (20 hours).

  5. To prompt users to grant the app consent to use their account, select the Prompt for user consent check box.

  6. To enable the use of refresh tokens, select the Enable refresh tokens check box.

  7. After reading the Non-Commercial Developer Agreement, select the I accept the Non-Commercial Developer Agreement check box stating you agree to the terms.

  8. Click Register.

Refer this - https://docs.valence.desire2learn.com/basic/oauth2.html

After Registering the application click the name of the application for which we want to get the credentials and from that we can get ClientID and Client Secret for the secret fields.

Steps to revoke the content from an authenticated app

  1. Navigate to My Settings widget

  2. Click Account Settings

  3. In that application settings, Click Manage applications registered with OAuth 2.0

  4. On the Manage Applications page, click the app that you want to revoke authentication from

  5. Click Remove App.

Things to keep in mind

  1. API Rate Limiting: Brightspace API calls include additional data in the response header to indicate the remaining number of credits available within the current minute, which implies there’s a limit to the number of API calls that can be made.

    • Solution: Manage the number of API calls and monitor the X-Rate-Limit-Remaining header to stay within the rate limits. If the limit is exceeded, the information will be available through the Brightspace System Log.

    • Refer to this - https://community.d2l.com/brightspace/kb/articles/1460-brightspace-api-rate-logging

  2. User Context Limitations: Every API call happens within a calling user context, and the LMS limits the functionality extended to your call depending on the permissions that user would have.

    • Solution: Ensure that the user context for the API call has the necessary permissions to perform the desired actions.

    • Refer to this - https://docs.valence.desire2learn.com/basic/firstlist.html

  3. Character Limitations: There are character limits within Brightspace when editing fields, which can prevent saving until the number of characters is under the specified limit.

    • Solution: Be mindful of character limits when designing the chatbot’s responses to ensure compatibility with Brightspace’s character limitations.

    • Refer to this - https://community.d2l.com/brightspace/kb/articles/18831-character-limitations-within-brightspace
