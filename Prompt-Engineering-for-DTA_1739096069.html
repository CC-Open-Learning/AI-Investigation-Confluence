Prompt Engineering is one of the most essential techniques in development using Large Language Models (LLMs). Designing prompts that guide the model in generating the expected responses and delivering more accurate results plays a critical role in shaping LLM-based applications' user experience (UX).

Through prompt engineering, developers can exert more precise control over the model’s output, enabling flexible responses to various user needs. This methodology ultimately enhances both the reliability and the usability of applications.

Use of Prompt Engineering in DTA Development

In the development of the Digital Teaching Assistant (DTA), two key types of prompt engineering were applied, each serving a specific purpose:

1. Prompts for validating correct answers

This prompt is designed to strictly evaluate whether a user's answer is functionally or conceptually correct, depending on the type of question.

Key characteristics:

  • Binary evaluation: The assistant only returns 'true' or 'false'.

  • Supports two question types:

    • Programming questions:

      • Allows for variation in implementation (different syntax, loop types, etc.)

      • Focuses on functional correctness and behavior, not exact code match.

      • Accepts minor formatting or syntax differences if the output is correct.

    • General (conceptual) questions:

      • Evaluates whether the core meaning of the user’s answer aligns with the expected answer.

      • Accepts rephrased responses as long as they contain the correct idea or fact.

The AI is explicitly instructed not to generate questions, alternative answers, or feedback, and to act only as a strict correctness checker.

2. Prompts for providing hints

This prompt helps the AI assistant generate non-revealing, encouraging hints to guide users toward the correct answer.

Key characteristics:

  • The assistant acts like a kind, supportive guide.

  • Considers:

    • The current answer

    • The previous answers

    • The number of attempts

  • Hints:

    • Should encourage logical reasoning and progress.

    • Must not reveal the correct answer.

    • It should be simple and informative, gently nudging the user in the right direction.

  • If the user has made 5 or more attempts, the assistant provides a slightly more specific hint to help them get unstuck, without disclosing the solution.

This prompt ensures that the assistant provides a scaffolded learning experience, supporting the user’s independent problem-solving.

Potential Issues

  1. LLM Model Selection and Service Changes

    1. Since S24, we have tested various LLM models and selected the Mistral model(mixtral-8x7b-32768) that best suited our system.

    2. However, in mid-March 2025, the LLM service we were using was discontinued. We are currently using the free version of Gemma(gemma2-9b-it), but we must always keep in mind that the service could be terminated at any time.

  2. Differences in Prompt Engineering Results

    1. The results of prompt engineering vary depending on the LLM model. After testing multiple models using the same prompt, some models provided concise responses, while others output lengthy and detailed explanations.

    2. Therefore, prompt engineering should be positioned as the final development process after selecting the LLM model, and optimization should be done according to the characteristics of the model.

  3. Model Accuracy Issues

    1. In the prompt_evaluation, the user’s answers are evaluated. However, due to model changes or ambiguous responses, errors can occur. In the case of coding problems, functional accuracy is evaluated, but there may be a lack of understanding regarding specific implementation methods. Therefore, updates must be made immediately when any issues are discovered.
