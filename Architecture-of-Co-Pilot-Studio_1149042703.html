AI Investigation : Architecture of Co-Pilot Studio

Created by Aerish Gaba (Unlicensed), last modified on Jun 03, 2024

This document is created to record the architecture of the copilot studio sourced from https://learn.microsoft.com/en-us/microsoft-copilot-studio/ .

The architecture solely depends on the feature of the copilot studio called Plugins.

  • Plugins - A reusable piece of code to perform a specific/designated task of a co-pilot.

for example - To send a pre- generated email after approval, Perform action by querying data in Dataverse.

  • Created in Plugin Authoring Section where user can create/edit plugins using a simplified GUI.

  • Once created the plugins can be used in many co-pilots to the authorized users as they are stored in a registry.

Above is the workflow of how the plugin cycle.

Creator Layer - Layer where user create/edit the plugin.

There are 4 types of plugins a user can create :-

  1. Prompt Plugin - Plugin where the copilot answers the user's queries and respond using Natural Language Processing (NLP) and Natural Linguistic Generation (NLG).

  2. Flow - Plugin where copilot executes multi step workflows using Power Automate.

  3. Connector - Plugin where copilot connects to external systems or data sources like GitHub, Zendesk, Salesforce or SAP using power platform connectors which makes it possible to retrieve or update data in real-time. https://learn.microsoft.com/en-us/microsoft-copilot-studio/copilot-plugins-overview

  4. Topic - Where copilot answers a specific user utterance or queries like store opening hours.

Plugin Registry - Stores/Manages plugin metadata and executes information.

Here Admin layer uses Microsoft admin centre to choose integrated apps available for tenants.

it allows, deploy or block the apps for specific or all users.

Bot Framework :- Key Concepts for composer/user.

  • Fallback Option - In addition to generative answers, we can use AI general knowledge to allow your copilot to find and present information in response to your customer's questions. General knowledge saves you from needing to manually author multiple topics, which might not even address all your customer's questions.
