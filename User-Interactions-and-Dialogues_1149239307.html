AI Investigation : User Interactions and Dialogues

Created by Arsh Chauhan, last modified on Jun 04, 2024

This document contains the planning and design of user interaction flows and dialogues for the eConestoga Chatbot. The objective is to create a intuitive, efficient and user-friendly chatbot that can assist the students, professors and also faculty members with various queries and tasks which are related to the Conestoga College services with the help of Microsoft Co-pilot Studio. This includes designing sample dialogues, identifying intents and entities for the chatbot, and also ensuring best practices in user direction.

Main Objectives

  • To design effective user interaction flows for the eConestoga Chatbot

  • To Create a sample dialogues for common use cases

  • To Identify and list the intents and entites that chatbot should recognize

  • To document the interaction design in a detailed and comprehensive manner.

Best Practices for Chatbot Dialogues and User Interaction

Key Principles

  1. Clarity and Simplicity - Ensure that the bot should communicate clearly and concisely and it should use NLP using Co-pilot studio to make sure that the responses should be in User friendly language.

  2. Contextual Awareness - The chatbot should be able to maintain context within conversations to provide relevant responses. For Example on asking for the assignment deadline it should give a straight answer instead of giving the links to the assignment tab.

  3. Error Handling - There should be recovery paths when the chatbot fails to understand user input. For example if there is a user input which is not clearly understandable by the bot it should be able to ask the user to use a better language so that chatbot can assist with proper understanding of the user input.

  4. Consistency - Use Consistent tone and style through the interactions.

Best Practices

  1. Greetings and Introduction - Start with a good friendly gesture and a brief introduction of the chatbot’s capabilities. For Example it could be like this “Hello I am Condor’s AI and i am here to assist you with your course feel free to ask questions like when is my assignment due or how much did i score in midterms or any other question.

  2. Follow-up Questions - Chatbot handling follow up questions related to the old query can help as sometimes student can be confused by the response they should be able to ask further questions from the bot.

  3. Acknowledgement and Empathy - Acknowledge users inputs and show them empathy when users are expressing frustrations or confusions

  4. Confirmation and Clarification - Confirm users inputs then giving them the reply can help in informing the user that output is going to be of the query which bot was able to understand from the User Input if its not right bot should be able to give the user option to clarify themselves.

Sample Dialogues and Interaction Flows

Common Use Cases

  • Answering FAQs - Providing answers to the most frequently asked questions about courses, admissions, schedules, etc.

  • Course Information - Providing details about course offerings, schedules and prerequisites. This can include passing criteria of a particular program and other things too.

  • Administrative Support - Helping with administrative tasks like ONE Card information, fees payments, program handbook support and other administrative support too.

  • Technical Support: Assisting with technical issues related to the eConestoga platform. This should be able to help users to clear all the technical doubts such as “How to access student portals” and other technical part too

Sample Dialogues

Use Case: Answering FAQs

  • Student: "What are the library hours?"

  • Chatbot: "The library is open from 8 AM to 4:30 PM, Monday through Friday, and it is closed on weekends. Is there anything else you need help with?"

Use Case: Course Information

  • Student: "Can you tell me about the Bachelors of Computer Science program?"

  • Chatbot: "The Bachelor of Computer Science (Honours) (BCS) degree is a four year program, including three paid co-op work terms, which will prepare graduates for a number of progressive job opportunities as high quality software developers. It is a 4 year course with prerequisites in basic statistics and programming. Would you like to know the schedule for this course?"

  • Chatbot: "The Data Science course covers topics such as data analysis, machine learning, and data visualization. It is a 3-credit course with prerequisites in basic statistics and programming. Would you like to know the schedule for this course?"

Use Case: Event Information

  • Professor: "When is the Assignment 3 due of Programming Principles?"

  • Chatbot: "The next assignment is due on Friday at 11:59 PM this week. Do you need any other information?"

Use Case: Technical Support

  • Faculty Member: "I'm having trouble logging into my account."

  • Chatbot: "I'm sorry to hear that. Can you provide more details about the issue you're facing? For example, are you getting an error message?"

Use Case: Administrative Support

  • Student: "How do I register for classes?"

  • Chatbot: "You can register for classes through the student portal. Go to the 'Course Registration' section and follow the instructions. If you need further assistance, I can guide you through the process."
